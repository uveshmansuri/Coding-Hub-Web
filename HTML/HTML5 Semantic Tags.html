HTML5 Semantic Tags

Semantic Tags

HTML5 introduced semantic tags to improve the structure and meaning of web content. These tags describe the role of the content within them, making it easier for search engines and developers to understand the layout of a webpage.

Benefits of Semantic Tags

  • Improves the readability of code.
  • Enhances SEO (Search Engine Optimization).
  • Facilitates better accessibility for assistive technologies.
  • Makes the structure of a webpage more meaningful.

Common HTML5 Semantic Tags

<header> Tag

The <header> tag defines the header of a webpage or a section. It typically contains introductory content or navigation links.

Example

header {
    background: #0078d7;
    color: #fff;
    padding: 20px 10px;
    text-align: center;
}                
<header>
Heading
</header>

Output

Heading

<article> Tag

The <article> tag defines independent, self-contained content, such as blog posts, news articles, or user comments.

Example

article {
    background: #f4f4f4;
    padding: 20px;
    margin: 10px 0;
    border: 1px solid #ddd;
}
<article>
    This is an article.
</article>

Output

This is an article.

<nav> Tag

The <nav> tag defines a navigation section in a document. It typically contains links to other parts of the website.

Example

nav {
    background: #ffe8e8;
    color: #fff;
    padding: 10px;
}                
<nav>
    <ul>
        <li><a href="#">Home</a></li>
        <li><a href="#">About</a></li>
        <li><a href="#">Contact</a></li>
    </ul>
</nav>

Output

  • Home
  • About
  • Contact

<section> Tag

The <section> tag defines a thematic grouping of content, typically with a heading.

Example

section {
    background: #e6e6e6;
    padding: 20px;
    margin: 10px 0;
    border-radius: 5px;
}
<section>
    This is a section.
</section>

Output

This is a section.

<main> Tag

The <main> tag represents the main content of a document, excluding repetitive content like headers and footers.

Example

main {
    background: #e9ecef;
    padding: 20px;
}                
<main>
    This is the main content.
</main>

Output

This is the main content.

<article> Tag

The <article> tag defines independent, self-contained content in a document, such as blog posts, articles, or forum posts.

Example

article {
    background: #00fff2;
    border: 1px solid #ccc;
    padding: 15px;
    margin-bottom: 10px;
}                
<article>
    This is an article.
</article>

Output

This is an article.

<aside> Tag

The <aside> tag defines content that is related to the main content but is not part of it. It is often used for sidebars or callout boxes.

Example

aside {
    background: #00ffff;
    border: 1px solid #ddd;
    padding: 15px;
    margin: 10px;
}                
<aside>
    This is an aside.
</aside>

Output

This is an aside.

<figure> and <figcaption> Tags

The <figure> tag is used to group media elements like images and their captions. The <figcaption> tag provides a caption for the figure.

Example

figure {
    text-align: center;
    margin: 10px 0;
}                
<figure>
    <img src="../Images/fc.jpeg" alt="Example Image" width="100%" />
    <figcaption>This is a caption.</figcaption>
</figure>

Output

<footer> Tag

The <footer> tag defines the footer of a webpage or a section, typically containing copyright, legal, or contact information.

Example

footer {
    background: #333;
    color: #fff;
    padding: 20px;
    text-align: center;
}
<footer>
    This is a footer.
</footer>

Output

This is a footer.
