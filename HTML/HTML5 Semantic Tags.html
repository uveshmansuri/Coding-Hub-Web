HTML5 Semantic Tags

What Are Semantic Tags?

HTML5 introduced semantic tags to improve the structure and meaning of web content. These tags describe the role of the content within them, making it easier for search engines and developers to understand the layout of a webpage.

Benefits of Semantic Tags

  • Improves the readability of code.
  • Enhances SEO (Search Engine Optimization).
  • Facilitates better accessibility for assistive technologies.
  • Makes the structure of a webpage more meaningful.

Common HTML5 Semantic Tags

<header>

The <header> tag defines the header of a webpage or a section. It typically contains introductory content or navigation links.

<nav>

The <nav> tag is used to define a block of navigation links.

Example:

<nav>
    <ul>
        <li><a href="#home">Home</a></li>
        <li><a href="#about">About</a></li>
        <li><a href="#services">Services</a></li>
    </ul>
</nav>

<main>

The <main> tag is used to define the main content of a webpage. This section should contain the unique content relevant to the page.

<section>

The <section> tag represents a thematic grouping of content, typically with its own heading.

Example:

<section>
    <h2>About Us</h2>
    <p>We provide the best services to our customers.</p>
</section>

<article>

The <article> tag represents a self-contained piece of content, such as a blog post, news article, or forum entry.

<aside>

The <aside> tag is used for content that is related but separate from the main content, such as sidebars, quotes, or advertisements.

<footer>

The <footer> tag defines the footer for a section or a webpage. It usually contains information like copyright notices, contact info, or links.
