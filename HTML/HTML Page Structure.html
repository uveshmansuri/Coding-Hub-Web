HTML Page Structure

Introduction

An HTML page structure serves as the foundation of any web document. It organizes content and metadata, enabling browsers to display the page correctly. The structure follows a hierarchy, starting from the document type declaration to the arrangement of the head and body sections.

Basic Structure of an HTML Page

The basic structure of an HTML page is standardized for consistency and readability. It includes:

  • Document Type Declaration (<!DOCTYPE html>): Defines the document as an HTML5 document.
  • <html> Element: The root element that contains the entire HTML document.
  • <head> Element: Contains metadata, styles, and links to external resources.
  • <body> Element: Contains all the visible content displayed on the webpage.

The general structure looks like this:

<!DOCTYPE html>
<html lang="en">
    <head>
        <title>Page Title</title>
        <meta charset="UTF-8">
    </head>
    <body>
        Content goes here
    </body>
</html>

Head Section

The <head> section contains information about the document that is not displayed directly on the page. It typically includes:

  • <title>: Sets the title of the document, displayed in the browser's title bar or tab.
  • <meta>: Provides metadata about the document, such as character encoding and author.
  • <link>: Links to external stylesheets or resources.
  • <style>: Adds internal CSS styles.
  • <script>: Links or embeds JavaScript for interactivity.

Example:

<head>
    <title>My Website</title>
    <meta charset="UTF-8">
    <meta name="description" content="An example webpage">
    <link rel="stylesheet" href="styles.css">
</head>

Body Section

The <body> section contains the visible content displayed to users, such as text, images, links, and other elements.

Common elements inside the <body> include:

  • <header>: Displays the header section, often containing navigation menus or logos.
  • <nav>: Provides a navigation bar for linking to other sections or pages.
  • <section>: Groups related content into blocks.
  • <article>: Represents self-contained content, such as blog posts or news articles.
  • <footer>: Displays footer information like copyright or contact details.

Example of a simple <body> structure:

<body>
    <header>Welcome to My Website</header>
    <nav>
        <a href="#section1">Section 1</a>
        <a href="#section2">Section 2</a>
    </nav>
    <main>
        <section id="section1">Content for Section 1</section>
        <section id="section2">Content for Section 2</section>
    </main>
    <footer>Footer Content</footer>
</body>
