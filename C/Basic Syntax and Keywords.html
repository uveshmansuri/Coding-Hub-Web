Basic Syntax, Keywords, and Identifiers in C Language

Introduction

The C programming language follows a specific syntax that dictates how programs should be written. These rules are essential to ensure that the code is correctly interpreted and executed by the compiler.

Basic Syntax of a C Program

A simple C program consists of the following components:

  • Preprocessor directives (e.g., `#include`).
  • The `main()` function, where the execution starts.
  • Statements and expressions within the `{}` braces.
  • Semicolon (`;`) at the end of every statement.

Example of a Basic C Program

#include <stdio.h>  // Preprocessor directive

int main() {            // Start of the main function
    printf("Hello, World!\n");  // Print statement
    return 0;           // Return statement
}                       // End of the main function

Explanation: This program prints "Hello, World!" to the console.

Keywords in C

Keywords are reserved words in C that have predefined meanings. They cannot be used as identifiers or variable names. C has 32 standard keywords:

Examples of Keywords:

auto, break, case, char, const, continue, default, do, double, else, enum, extern, float, for, goto, if, int, long, register, return, short, signed, sizeof, static, struct, switch, typedef, union, unsigned, void, volatile, while

Identifiers in C

Identifiers are the names used to identify variables, functions, or any user-defined element in a C program. They must follow specific rules:

  • Must start with a letter (A-Z or a-z) or an underscore (_).
  • Can contain letters, digits (0-9), and underscores.
  • Cannot be a keyword.
  • Case-sensitive (e.g., `Variable` and `variable` are different).

Examples of Valid and Invalid Identifiers

Valid Identifiers:

  • myVariable
  • _temp
  • count123

Invalid Identifiers:

  • 123count (Cannot start with a digit)
  • for (Keyword)
  • my-variable (Contains a hyphen, which is not allowed)

Important Points to Remember

  • C is case-sensitive, so `Variable` and `variable` are treated as different identifiers.
  • Keywords cannot be redefined or used as identifiers.
  • Follow proper naming conventions to make the code more readable.

Conclusion

Understanding the syntax, keywords, and identifiers is the foundation of programming in C. Following these rules ensures that your program compiles and runs correctly.
