Comments in C

Introduction

In C programming, comments are used to explain the code and make it more readable. They are ignored by the compiler and have no effect on the execution of the program. Comments are crucial for documenting code and aiding other developers (or yourself) in understanding the program later.

Types of Comments in C

C supports two types of comments:

  1. Single-line Comments
  2. Multi-line Comments

1. Single-line Comments

Single-line comments start with //. Everything after // on the same line is treated as a comment.

Example:

#include <stdio.h>

int main() {
    // This is a single-line comment
    printf("Hello, World!\\n"); // This prints a message
    return 0;
}

In this example:

  • The first comment explains the program's purpose.
  • The second comment describes the printf statement.

2. Multi-line Comments

Multi-line comments begin with /* and end with */. They can span multiple lines.

Example:

#include <stdio.h>

int main() {
    /* This is a multi-line comment.
       It can span multiple lines.
       Use it for detailed explanations. */
    printf("Hello, World!\\n");
    return 0;
}

In this example, the multi-line comment explains the program over several lines.

Best Practices for Using Comments

  • Use comments to explain why a particular block of code is written, not how it works (the code should be self-explanatory).
  • Avoid overusing comments; keep them concise and relevant.
  • Use single-line comments for short notes and multi-line comments for detailed descriptions.

Examples

Example 1: Describing Variables

#include <stdio.h>

int main() {
    int num = 10; // This variable stores a number
    printf("Number: %d\\n", num);
    return 0;
}

Example 2: Explaining a Loop

#include <stdio.h>

int main() {
    /* Loop to print numbers from 1 to 5 */
    for (int i = 1; i <= 5; i++) {
        printf("%d\\n", i); // Print the current number
    }
    return 0;
}

Output for Example 2:

1
2
3
4
5

Key Points

  • Comments improve code readability and maintainability.
  • They are ignored by the compiler, so they do not affect the program's performance.
  • Always ensure comments are meaningful and not redundant.
