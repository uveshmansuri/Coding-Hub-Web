Class and Object in Java

In Java, Class and Object are fundamental concepts of Object-Oriented Programming (OOP). They are the building blocks for creating reusable, modular code and help in modeling real-world entities in software.

Class in Java

A Class is a blueprint or template for creating objects. It defines a type of object according to the data it stores (called attributes or fields) and the operations it can perform (called methods). A class can contain constructors, fields, and methods to define the behavior of the objects it creates.

Components of a Class:

  • Fields/Attributes: Variables that store data for the object.
  • Methods: Functions that define the behavior of the object.
  • Constructor: A special method used to initialize objects.

Syntax of a Class:


class ClassName {
    // Fields (attributes)
    int attribute1;
    String attribute2;

    // Constructor (optional)
    public ClassName(int attribute1, String attribute2) {
        this.attribute1 = attribute1;
        this.attribute2 = attribute2;
    }

    // Methods (functions)
    public void display() {
        System.out.println("Attribute1: " + attribute1);
        System.out.println("Attribute2: " + attribute2);
    }
}
  • Field/Attribute: int attribute1, String attribute2
  • Constructor: public ClassName(int attribute1, String attribute2)
  • Method: public void display()

Object in Java

An Object is an instance of a class. When a class is defined, no memory is allocated. Memory is allocated only when an object of that class is created. An object has state (attributes/fields) and behavior (methods).

Creating Objects:


ClassName objectName = new ClassName(parameter1, parameter2);
  • ClassName: Name of the class.
  • objectName: Reference variable that will hold the memory address of the created object.
  • new ClassName(parameter1, parameter2): This is the constructor of the class, which creates and initializes the object.

Detailed Example: Class and Object

Step 1: Define the Class


class Car {
    // Fields (attributes)
    String make;
    String model;
    int year;

    // Constructor to initialize the object
    public Car(String make, String model, int year) {
        this.make = make;
        this.model = model;
        this.year = year;
    }

    // Method to display car details
    public void displayDetails() {
        System.out.println("Make: " + make);
        System.out.println("Model: " + model);
        System.out.println("Year: " + year);
    }
}
  • Fields: String make, String model, int year
  • Constructor: Initializes the values of make, model, and year when a new Car object is created.
  • Method: displayDetails() displays the details of the car.

Step 2: Create Objects and Use the Class


public class Main {
    public static void main(String[] args) {
        // Creating objects of the Car class
        Car car1 = new Car("Toyota", "Corolla", 2020);  // Object car1
        Car car2 = new Car("Honda", "Civic", 2022);    // Object car2

        // Calling method using objects
        car1.displayDetails();
        System.out.println();
        car2.displayDetails();
    }
}

Output:

Make: Toyota
Model: Corolla
Year: 2020

Make: Honda
Model: Civic
Year: 2022

Explanation of the Example

Class Definition (Car)

The Car class has three fields (make, model, year), a constructor that initializes these fields, and a method (displayDetails) that prints the car's information.

Object Creation

In the main method, two objects (car1 and car2) are created using the Car class constructor. 

new Car("Toyota", "Corolla", 2020)

This creates an object and initializes the fields make, model, and year.

Calling Methods

We call the displayDetails() method on each object (car1 and car2) to display the car details.

Key Points about Classes and Objects in Java

  • Class as a Blueprint: A class defines a structure, but no memory is allocated until an object is created.
  • Object as an Instance: An object is an instance of a class. Each object has its own state (attributes) and can perform actions (methods).
  • Constructor: A constructor is used to initialize the object's state when it is created. It has the same name as the class and does not have a return type.
  • Methods: Methods define the behavior of the class. Objects use these methods to perform actions on their data (attributes).
  • Accessing Objects: Objects are accessed via reference variables. They can be passed around in methods or returned from functions.

Encapsulation with Objects

Objects support encapsulation, which means the internal state (attributes) of an object is hidden from the outside, and access is controlled via public methods (getters/setters). This ensures data security and controlled access.

For example, to protect the year attribute, you can use:

class Car {
    private int year;  // Private field

    public void setYear(int year) {  // Setter
        if (year > 1900 && year <= 2024) {
            this.year = year;
        } else {
            System.out.println("Invalid year");
        }
    }

    public int getYear() {  // Getter
        return year;
    }
}

In this case, the year field is private, and can only be accessed or modified using the getYear() and setYear() methods, providing control over how it is accessed.

Summary

In summary, a Class is a template that defines the structure and behavior of objects, while an Object is an instance of that class. Classes contain attributes and methods, and objects use those methods to operate on their data. Together, they represent the core concepts of Object-Oriented Programming (OOP) in Java, enabling modular, reusable, and organized code.
