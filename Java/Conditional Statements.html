Conditional Statements in Java

Introduction

Conditional statements allow a program to make decisions and execute specific blocks of code based on certain conditions. These statements help control the flow of execution in a program.

Types of Conditional Statements

  1. if Statement
  2. if-else Statement
  3. if-else-if Ladder
  4. Nested if Statement
  5. switch Statement

1. if Statement

The if statement executes a block of code only if a specified condition is true.

Syntax:


if (condition) {
    // code to be executed if condition is true
}

Example:


int number = 10;

if (number > 0) {
    System.out.println("The number is positive.");
}

Explanation:

The condition number > 0 is evaluated:

  • Here, number is 10, so 10 > 0 evaluates to true.
  • Since the condition is true, the code inside the if block is executed.

Output:

The number is positive.

2. if-else Statement

The if-else statement provides an alternative block of code to execute when the condition is false.

Syntax:


if (condition) {
    // code to be executed if condition is true
} else {
    // code to be executed if condition is false
}

Example:


int number = -5;

if (number > 0) {
    System.out.println("The number is positive.");
} else {
    System.out.println("The number is not positive.");
}

Explanation:

The condition number > 0 is evaluated:

  • Here, number is -5, so -5 > 0 evaluates to false.
  • Since the condition is false, the else block is executed.

Output:

The number is not positive.

3. if-else-if Ladder

The if-else-if ladder is used when multiple conditions need to be tested sequentially. Only the first true condition is executed.

Syntax:


if (condition1) {
    // code to be executed if condition1 is true
} else if (condition2) {
    // code to be executed if condition2 is true
} else {
    // code to be executed if all conditions are false
}

Example:


int number = 0;

if (number > 0) {
    System.out.println("The number is positive.");
} else if (number < 0) {
    System.out.println("The number is negative.");
} else {
    System.out.println("The number is zero.");
}

Explanation:

The first condition number > 0 is evaluated:

  • Here, number is 0, so 0 > 0 evaluates to false.
  • The second condition number < 0 is evaluated: 0 < 0 also evaluates to false.
  • Since both conditions are false, the else block is executed.

Output:

The number is zero.

4. Nested if Statement

A nested if statement is an if statement inside another if or else block.

Syntax:


if (condition1) {
    if (condition2) {
        // code to be executed if both condition1 and condition2 are true
    }
}

Example:


int number = 15;

if (number > 0) {
    if (number % 2 == 0) {
        System.out.println("The number is positive and even.");
    } else {
        System.out.println("The number is positive and odd.");
    }
}

Explanation:

  • The first condition number > 0 is true because 15 > 0.
  • Inside the outer if, the nested condition number % 2 == 0 is false.
  • The code inside the nested else block is executed.

Output:

The number is positive and odd.

5. switch Statement

The switch statement evaluates an expression and executes a block of code corresponding to the matching case label.

Syntax:


switch (expression) {
    case value1:
        // code to be executed if expression == value1
        break;
    // other cases
    default:
        // code to be executed if none of the cases match
}

Example:


int day = 3;

switch (day) {
    case 1:
        System.out.println("Monday");
        break;
    case 2:
        System.out.println("Tuesday");
        break;
    case 3:
        System.out.println("Wednesday");
        break;
    default:
        System.out.println("Invalid day");
}

Explanation:

  • The expression day matches case 3.
  • The code inside case 3 is executed.
  • The break prevents execution from falling through to other cases.

Output:

Wednesday

Difference Between if-else and switch

Aspect if-else switch
Usage Used for checking conditions with relational or logical operators. Used for checking a single variable against multiple constant values.
Conditions Can evaluate complex conditions using relational and logical operators. Can only evaluate expressions with constant values (e.g., int, char, string, enums).
Efficiency Less efficient for a large number of conditions because all conditions are checked sequentially. More efficient for a large number of cases because it uses a jump table internally.
Flexibility More flexible as it supports conditions, ranges, and logical operators. Limited flexibility as it only supports equality checks against constants.
Default Case No default case; conditions must be explicitly written in the else block. Has a default case to handle unmatched values.

Conclusion: Use if-else for complex conditions and switch for simpler value-based checks.

Key Points to Remember

  • if vs. switch: Use if for complex conditions or ranges. Use switch for fixed, discrete values.
  • Curly Braces: Always use {} even for single statements in if, else, or case for clarity and to avoid errors.
  • Break in switch: The break statement prevents fall-through. Omitting break causes the execution to continue to the next case.
  • default Case: Always include a default case in switch to handle unexpected values.
