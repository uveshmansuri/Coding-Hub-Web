Polymorphism in C++

Polymorphism?

Polymorphism is one of the key concepts of Object-Oriented Programming (OOP) in C++. The term polymorphism is derived from Greek, meaning "many forms." In C++, polymorphism allows a single function, operator, or object to behave differently based on its context.

Types of Polymorphism:

  • Compile-time Polymorphism: Also known as static polymorphism, it is resolved during compilation. Examples include function overloading and operator overloading.
  • Runtime Polymorphism: Also known as dynamic polymorphism, it is resolved at runtime. It is implemented using inheritance and virtual functions.

Runtime Polymorphism in C++

Runtime polymorphism is achieved through the use of inheritance and virtual functions. It allows a base class pointer or reference to call a method of a derived class. This behavior is determined during runtime, enabling dynamic method dispatch.

Key Characteristics:

  • Uses virtual functions in the base class to achieve polymorphism.
  • Enables overriding of base class functions in derived classes.
  • Achieved using a pointer or reference to the base class.
  • Supports flexibility and extensibility in code.

Example of Runtime Polymorphism:

#include <iostream>
using namespace std;

// Base class
class Shape {
public:
    virtual void draw() {
        cout << "Drawing a generic shape" << endl;
    }
};

// Derived class: Circle
class Circle : public Shape {
public:
    void draw() override {
        cout << "Drawing a circle" << endl;
    }
};

// Derived class: Rectangle
class Rectangle : public Shape {
public:
    void draw() override {
        cout << "Drawing a rectangle" << endl;
    }
};

int main() {
    Shape* shapePtr;  // Base class pointer

    Circle circle;
    Rectangle rectangle;

    shapePtr = &circle;
    shapePtr->draw(); // Calls Circle's draw()

    shapePtr = &rectangle;
    shapePtr->draw(); // Calls Rectangle's draw()

    return 0;
}

Output:

Drawing a circle
Drawing a rectangle

Compile-time Polymorphism in C++

Compile-time polymorphism occurs when the decision about which function to call is made at compile-time. It is achieved through function overloading and operator overloading.

Key Characteristics:

  • Does not involve inheritance or virtual functions.
  • Improves performance as decisions are made during compilation.
  • Allows multiple functions or operators to perform differently based on their input parameters.

Example of Function Overloading:

#include <iostream>
using namespace std;

class Math {
public:
    // Function to add two integers
    int add(int a, int b) {
        return a + b;
    }

    // Function to add three integers
    int add(int a, int b, int c) {
        return a + b + c;
    }
};

int main() {
    Math math;
    cout << "Sum of 2 numbers: " << math.add(10, 20) << endl;
    cout << "Sum of 3 numbers: " << math.add(10, 20, 30) << endl;
    return 0;
}

Output:

Sum of 2 numbers: 30
Sum of 3 numbers: 60

Difference between Runtime and Compile-time Polymorphism:

Aspect Compile-time Polymorphism Runtime Polymorphism
Definition Resolved at compile-time using function or operator overloading. Resolved at runtime using inheritance and virtual functions.
Mechanism Function signature determines behavior at compile-time. Virtual table (vtable) and dynamic binding determine behavior at runtime.
Performance Faster due to compile-time resolution. Slower due to dynamic method dispatch.
Flexibility Less flexible and limited to parameter variations. Highly flexible due to dynamic method overriding.
Use Cases Function overloading, operator overloading. Inheritance and overriding virtual functions.

Conclusion:

Polymorphism is a powerful feature of C++ that enhances code reusability and flexibility. While compile-time polymorphism offers better performance, runtime polymorphism provides adaptability and is ideal for applications requiring dynamic behavior.
