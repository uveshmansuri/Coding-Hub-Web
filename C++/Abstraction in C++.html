Abstraction in C++

What is Abstraction?

Abstraction in C++ is a core concept of object-oriented programming that focuses on hiding implementation details and showing only essential features to the user. It helps to manage complexity by segregating what an object does from how it does it.

For example, when using a car, we interact with the steering wheel, accelerator, and brakes without worrying about the internal mechanics of how they work.

How to Achieve Abstraction in C++?

Abstraction in C++ is achieved using:

  • Abstract classes (classes containing pure virtual functions).
  • Access specifiers like private, protected, and public to restrict access to certain parts of a class.

Example: Abstraction with Abstract Class

#include <iostream>
using namespace std;

// Abstract class
class Shape {
public:
    // Pure virtual function
    virtual void draw() = 0;

    void setColor(string c) {
        color = c;
    }

    string getColor() {
        return color;
    }

private:
    string color; // Hidden detail
};

class Circle : public Shape {
public:
    void draw() override {
        cout << "Drawing a Circle" << endl;
    }
};

class Rectangle : public Shape {
public:
    void draw() override {
        cout << "Drawing a Rectangle" << endl;
    }
};

int main() {
    Shape* shape;

    Circle circle;
    shape = &circle;
    shape->setColor("Red");
    cout << "Circle Color: " << shape->getColor() << endl;
    shape->draw();

    Rectangle rectangle;
    shape = &rectangle;
    shape->setColor("Blue");
    cout << "Rectangle Color: " << shape->getColor() << endl;
    shape->draw();

    return 0;
}

Output:

Circle Color: Red
Drawing a Circle
Rectangle Color: Blue
Drawing a Rectangle

Advantages of Abstraction

  • Reduces complexity by exposing only essential functionalities.
  • Improves code readability and maintainability.
  • Provides a clear separation between interface and implementation.
  • Helps in achieving modularity and reusability.

Abstraction vs Encapsulation

Though closely related, abstraction and encapsulation are distinct concepts:

Aspect Abstraction Encapsulation
Definition Hiding implementation details and showing only functionality. Wrapping data and functions into a single unit (class) and controlling access using access specifiers.
Focus On "what" an object does. On "how" the data is protected.
Mechanism Achieved using abstract classes and interfaces. Achieved using classes and access specifiers.

Conclusion

Abstraction is a powerful tool in C++ for managing complexity and focusing on the functionality provided by an object rather than its internal workings. By leveraging abstract classes and access specifiers, developers can create clean, modular, and maintainable code.
