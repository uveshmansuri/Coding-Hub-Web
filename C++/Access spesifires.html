Access Specifiers in C++

Access Specifiers

In C++, access specifiers are used to control the accessibility of class members (data members and member functions). The primary goal is to achieve encapsulation, which is a fundamental concept of Object-Oriented Programming (OOP). Encapsulation ensures that the internal state of an object is protected from unintended or harmful changes and can only be accessed or modified through well-defined interfaces (i.e., public methods).

The three main access specifiers are private, public, and protected.

Types of Access Specifiers

Private Access Specifier

The private access specifier restricts access to members only within the class. These members cannot be accessed from outside the class, including derived classes, unless through a public or protected function.

Private members are used to hide data and ensure that it is only modified through public or protected functions, enforcing data integrity and consistency.

Example of Private Access Specifier

class BankAccount {
private:
    double balance; // private data member

public:
    // Public member function to set balance
    void setBalance(double bal) {
        if (bal >= 0) {
            balance = bal;
        }
    }

    // Public member function to get balance
    double getBalance() {
        return balance;
    }
};
Explanation:
  • The balance variable is private, meaning it cannot be accessed directly from outside the class.
  • The setBalance() and getBalance() functions are public, allowing controlled access to the private balance.

Public Access Specifier

The public access specifier allows class members to be accessed from anywhere, including outside the class. Any part of the program can use public members, making them available to users of the class.

Public members are often used for functions that define the interface of the class, such as constructors, destructors, and methods that control object state.

Example of Public Access Specifier

class Rectangle {
public:
    double length;  // public data member
    double width;   // public data member

    double area() {
        return length * width; // Public method to calculate area
    }
};
Explanation:
  • The variables length and width are declared public, so they can be accessed and modified directly from outside the class.
  • The area() method is public, meaning it can be called from outside the class to calculate the area of a rectangle.

Protected Access Specifier

The protected access specifier is a middle ground between private and public access. Protected members can be accessed within the class and by derived classes (i.e., subclasses), but they cannot be accessed from outside the class hierarchy.

This access level is useful when you want to allow derived classes to access and modify base class members, but prevent external code from doing so.

Example of Protected Access Specifier

class Shape {
protected:
    double width;
    double height;

public:
    void setDimensions(double w, double h) {
        width = w;
        height = h;
    }
};

class Rectangle : public Shape {
public:
    double area() {
        return width * height; // Accessing protected members of the base class
    }
Explanation:
  • The variables width and height are declared protected in the base class Shape.
  • The derived class Rectangle has access to these protected members and can directly use them to calculate the area.
  • However, external code cannot directly access these protected members.

Default Access Specifiers

Default Access for Classes

If no access specifier is explicitly provided for members of a class, C++ uses the default access specifier. By default, members of a class are private, while members of a struct are public.

Example of Default Access

class Car {
    string brand; // private by default
public:
    void setBrand(string b) {
        brand = b; // Accessing private member through a public function
    }
};
Explanation:
  • The brand variable is private by default, as no access specifier is specified.
  • The setBrand() function is public and provides access to the private member brand.

Inheritance and Access Specifiers

Inheritance and Access Levels

Access specifiers play a significant role when dealing with inheritance. The accessibility of base class members in derived classes depends on both the member's access specifier and the inheritance type.

Example of Inheritance and Access Specifiers

class Base {
private:
    int privateVar; // Private member
protected:
    int protectedVar; // Protected member
public:
    int publicVar; // Public member
};

class Derived : public Base {
public:
    void accessMembers() {
        // privateVar is not accessible here, causes error
        // protectedVar is accessible here
        // publicVar is accessible here
    }
};
Explanation:
  • The privateVar is not accessible in the derived class.
  • The protectedVar and publicVar can be accessed in the derived class.

Summary

Access Specifier Accessibility Default for Class Members
Private Accessible only within the class and its member functions Yes (default for class members)
Public Accessible from anywhere No
Protected Accessible within the class and by derived classes No
