Constructor and Destructor in C++

Constructor and Destructor

In C++, a constructor and a destructor are special member functions of a class. A constructor is used to initialize objects of a class, while a destructor is used to clean up resources when an object is destroyed. These functions help manage the object's lifecycle and ensure proper memory and resource management.

Constructor in C++

A constructor is a special function that is automatically called when an object of the class is created. It is primarily used to initialize the object's data members and allocate resources as needed.

eatures of Constructors

  • Constructor name is the same as the class name.
  • Constructors do not have a return type.
  • They are automatically called when an object is instantiated.
  • Constructors can be overloaded (i.e., multiple constructors with different parameters).
  • Constructors can have default arguments, making them versatile in object creation.

Types of Constructors

There are three types of constructors in C++:

  • Default Constructor: A constructor that takes no arguments and initializes the object with default values.
  • Parameterized Constructor: A constructor that takes parameters and uses them to initialize object data members.
  • Copy Constructor: A constructor that creates a new object as a copy of an existing object of the same class.

Default Constructor Example

class Box {
public:
    int length, width;

    // Default Constructor
    Box() {
        length = 0;
        width = 0;
    }

    // Function to display values
    void display() {
        cout << "Length: " << length << ", Width: " << width << endl;
    }
};

int main() {
    Box box1;           // Default constructor called
    box1.display();     // Displays Length: 0, Width: 0
    return 0;
}
Explanation:
  • The default constructor is invoked when the object box1 is created.
  • The constructor sets the object's length and width to 0.
  • The display() function shows these values when called.

Parameterized Constructor Example

class Box {
public:
    int length, width;

    // Parameterized Constructor
    Box(int l, int w) {
        length = l;
        width = w;
    }

    void display() {
        cout << "Length: " << length << ", Width: " << width << endl;
    }
};

int main() {
    Box box2(10, 20);  // Parameterized constructor called
    box2.display();    // Displays Length: 10, Width: 20
    return 0;
}
Explanation:
  • The parameterized constructor initializes box2 with values 10 and 20 for length and width, respectively.
  • The values are displayed when display() is called.

Copy Constructor Example

class Box {
public:
    int length, width;

    // Parameterized Constructor
    Box(int l, int w) {
        length = l;
        width = w;
    }

    // Copy Constructor
    Box(const Box &b) {
        length = b.length;
        width = b.width;
    }

    void display() {
        cout << "Length: " << length << ", Width: " << width << endl;
    }
};

int main() {
    Box box1(10, 20);  // Parameterized constructor called
    Box box2 = box1;   // Copy constructor called
    box2.display();    // Displays Length: 10, Width: 20
    return 0;
}
Explanation:
  • The copy constructor is called when box2 is created as a copy of box1.
  • It initializes the new object with the values from the existing one, which are 10 and 20 for length and width.

Destructor in C++

A destructor is a special member function that is automatically called when an object goes out of scope or is explicitly deleted. The destructor is used to release resources that were allocated by the constructor or during the object's lifetime.

Features of Destructors

  • The destructor name is the same as the class name, preceded by a tilde (~).
  • Destructors do not take any parameters and cannot be overloaded.
  • A class can only have one destructor.
  • Destructors are called automatically when an object is destroyed (either goes out of scope or is explicitly deleted).

Example of Destructor

class Box {
public:
    int* ptr;

    // Constructor that allocates memory
    Box() {
        ptr = new int; // Dynamically allocated memory
        *ptr = 10;     // Assigning value
    }

    // Destructor to free dynamically allocated memory
    ~Box() {
        delete ptr; // Free memory when object is destroyed
    }

    void display() {
        cout << "Value: " << *ptr << endl;
    }
};

int main() {
    Box box1;           // Constructor called, memory allocated
    box1.display();     // Displays Value: 10
    // Destructor called when box1 goes out of scope
    return 0;
}
Explanation:
  • The constructor dynamically allocates memory for an integer using new.
  • The destructor ensures that the dynamically allocated memory is properly freed using delete when the object is destroyed, preventing memory leaks.

Constructor and Destructor with Objects

Constructors and destructors are called automatically when an object is created or destroyed. Here's how these special functions work in practice:

Example of Constructor and Destructor in Action

int main() {
    Box box1;           // Default constructor called
    Box box2(5, 10);    // Parameterized constructor called

    box1.display();
    box2.display();

    return 0;
} // Destructor automatically called here when box1 and box2 go out of scope
Explanation:
  • When box1 is created, the default constructor is called, initializing length and width to zero.
  • When box2 is created, the parameterized constructor is called, initializing the object with the specified values.
  • As the function main() finishes, destructors for both box1 and box2 are called automatically, cleaning up any resources if needed.

Destructor and Dynamic Memory Allocation

Destructors are especially useful when working with dynamic memory allocation. If an object creates resources such as dynamically allocated memory or file handles, the destructor is responsible for releasing them before the object is destroyed.

Example of Destructor Handling Dynamic Memory

class Data {
public:
    int* arr;
    int size;

    Data(int s) {
        size = s;
        arr = new int[size];  // Dynamically allocating memory
    }

    ~Data() {
        delete[] arr;  // Releasing memory when the object is destroyed
    }

    void fillData() {
        for (int i = 0; i < size; i++) {
            arr[i] = i * 10;
        }
    }

    void display() {
        for (int i = 0; i < size; i++) {
            cout << arr[i] << " ";
        }
        cout << endl;
    }
};
Explanation:
  • The constructor dynamically allocates an array of integers based on the specified size.
  • The destructor ensures that the allocated memory is properly freed when the object is destroyed, preventing memory leaks.

Conclusion

Constructors and destructors are essential components of C++ classes. They play a key role in object initialization and cleanup. By using constructors to initialize objects and destructors to release resources, C++ developers can create more robust and efficient code that manages resources effectively. Properly handling memory allocation and deallocation in destructors helps prevent memory leaks and ensures the proper functioning of programs.
